2016-9-29

  • Location for using my virtual machine: http://port.ac.uk/myvm
  • Learned how to create a webpage and upload it to my virtual machine.
  • <!DOCTYPE html> is used to declare a webpage using HTML5. This lets the browser know it is viewing an HTML document.
  • HTML 4.01 was based on SGML (standard generalised markup language), meaning <!DOCTYPE> referred to the page’s DTD (document type definition). The DTD declares how the markup language should be rendered on the browser according to specified rules.
  • For example, <!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01//EN" "http://www.w3.org/TR/html4/strict.dtd"> is HTML 4.01 strict, which does not allow deprecated elements.
  • HTML5 is not based on SGML, so <!DOCTYPE html> doesn’t refer to a DTD. It simply tells the browser to display an HTML5 document.

2016-11-17

CSS:

  • Basic syntax for CSS: Selector { attribute: value; }
  • For example: “p { color: white; }“ would colour all text inside a p-tag to white.
  • A period is used to select an element class. I.e: .classSelector
  • A hash sign is used to select an element id. I.e: #idSelector
  • The browser converts markup language & CSS of web document into DOM (Document Object Model). The document content and style is combined. It then displays the content of the DOM.

2016-11-24

Links:

  • Basic syntax: <a href=”URL”></a>
  • Absolute URLs (i.e. to other web sites) require the scheme (http://)
  • Relative URLs omit the scheme. Can link to pages in same directory (page.html), child directories (folder/page.html) and parent directories (../folder.html)
  • Target attribute: An option for <a> which specifies the window to open the linked page in.
  • target = “_self” Opens in current window (default)
  • target = “_blank” Opens in a new window (not encouraged, can hinder usability and accessibility of web content)
  • target = “framename” Opens in a framed name
  • Fragment links are used to scroll the page to a particular element on the page.
  • To do this, add an id tag to something, then make a relative url, but instead of href=“url.com”, use href=“#idName”
  • This can be done to scroll to a particular element on even a different page, using href=“url.com/#idName”
  • <nav>(HTML5): Acts as a container for links to other web pages. It removes the need to use a <div> tag with something such as class=”navigation” to define a section used for navigating the web.

2016-12-01

Images:

  • Hotlinking: involves linking images from other servers. Can save you some bandwidth, but risks the image being changed on server it comes from.
  • This gives a loss of control over the content being displayed. If the server changes the original image, then the image changes on your web page as well.
  • It is good practise to provide every image with an alt attribute. This is a short description on the image. This will be displayed instead should the image itself fail to be displayed.

Design:

  • The verb of the word is more important than the noun. Design defines how something works.
  • In design, everything is to have a purpose and a use. A minimalist and focused designer understands what the user wants and implements just that.
  • A design is finished when there is nothing left to take away from it.
  • Keeping a log of the design process is good practise. The design process can take a long time, so being able to look back is important for refreshing ideas and avoiding pitfalls.
  • In information design, information itself is the most important aspect. It is important to consider what the user wants, not just what looks good to designer.
  • Information is used to define short-term and long-term design goals.
  • I.e. understanding the competition helps to define the minimal design requirements that need to be met. Understanding the user ensures the design corresponds to their needs.

2017-02-07

Accessibility:

  • Law requires certain standards to be put in place to assist the impaired.
  • Likewise, websites should adhere to guidelines and principles which improve accessibility.
  • Doing so allows more people to use your website, which is good for both you and those whom rely on accessibility tools.
  • WCAG 2.0: Web content accessibility guidelines. It provides recommendations for designing more accessible web content. In particular, aria tags are helpful for providing semantic detail to web content (such as the “role” attribute to define the purpose of a piece of content, or “aria-haspopup” to notify accessibility tools that some content provides a popup).
  • Aria tags: these are added to html code to give each block of markup some semantic meaning. They help users with assistive technologies by defining points on the webpage designed to enhance accessibility.
  • HTML5 improves accessibility by providing built-in semantic tags to describe the content of a webpage (i.e. <nav>, <article>, <aside>).
  • This is better than simply using generic tags like <div> and <span> with only a developer-assigned class name to define the content, which is difficult or outright impossible for assistive technology to make use of.

2017-02-15

Videos:

  • Not all browsers support all video codecs.
  • To overcome this, providing copies of a video which use different codecs should be implemented to the webpage.
  • In HTML5, the <video> element is used to embed a video.
  • Within the video element, adding <source> allows you to define the source of the video and the type (its codec). Multiple <source> elements can be used, one for each video location and codec. This is useful for supplying copies of the same video with different codecs for different browsers.
  • I later found that to add controls to the video (i.e. pause, volume), the word “controls” should be added to the <video> element.
